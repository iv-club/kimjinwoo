JSP 문법

스크릿트 요소

선언문 - 전역변수 선언 및 메소드 선언에 사용(Service 메서드가 아니라 클래스 바디 쪽에 해당 코드가 바뀝니다.)

<%! %>

스크립트릿 - 프로그래밍 코드 기술에 사용(스크립트릿에서 선언된 변수는 지역변수(Service() 메서드 안에 선언되는 변수))

<% %>

표현식 - 화면에 출력할 내용 기술에 사용(응답 결과에 포함할 부분)

<%= %>

▶ 스크립트릿내에서 출력할 부분은 out.print를 이용하여 출력할 수 있습니다.

​

주석

▶ JSP 주석

    HTML 주석, JAVA 주석(//, /* */), JSP 주석(<%--로 시작해서 --%>로 끝나는 형태) 

    JSP 주석은 JSP페이지에서만 사용되기 때문에 서블릿으로 변환된지 않습니다.

    JAVA 주석은 JSP가 자바로 바뀔 때는 바뀌지만, 자바가 서블릿으로 실행할 때는 실행되지 않습니다.

​

▶ HTML 주석

    <!--로 시작해서 -->로 끝나는 형태

​

redirect

▶ 클라이언트가 서버에게 요청하면 서버가 일을 처리하고 클라이언트한테 응답을 하고 새로 요청할 곳을 알려주면 클라이언트가 다시 요청을 하는것입니다.(url 주소가 바뀜.)

서로 다른 요청이 두 번 발생되고 각각의 요청마다 request, response 객체가 한 번씩 만들어집니다.

​

forward

▶ 클라이언트가 서버에게 요청하면 서버쪽에서 그 요청을 혼자 처리하는 것이 아니라 다른 서버한테 나머지 처리를 위임하는 것입니다.(url 주소가 바뀌지 않음.)

클라이언트가 한 번 요청했을 때 만들어진 request, response 객체가 그대로 유지됩니다. request와 response를 다음 서버로 보내서 값을 전달합니다.

​

★ 클라이언트가 서버한테 요청을 하게 되면 WAS는 응답이 될 때까지 유지되는 request, response라는 객체를 만듭니다.
